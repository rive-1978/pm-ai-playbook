## Learning Loop

Before starting a new task, review existing rules and hypotheses for this domain in `06-system/learnings/`. Apply `rules.md` by default. Check if any `hypotheses.md` can be tested with today's work.

After completing a task, extract insights to the relevant domain folder:

- `knowledge.md` — facts, patterns, and observations
- `hypotheses.md` — possibilities that need more data
- `rules.md` — confirmed patterns (apply by default)

Maintain `06-system/learnings/INDEX.md` that routes to each domain folder. Create relevant domain folders, structure and update INDEX.md if it doesn't exist yet.

**Promotion:** When a hypothesis gets confirmed 5+ times, promote it to `rules.md`. When a rule is contradicted by new evidence, demote it to `hypotheses.md` with the contradiction noted.
